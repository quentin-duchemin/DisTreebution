Developer
=========

Thanks for considering contributing!

Guidelines
----------
- Open an issue describing the bug or desired enhancement.
- For code contributions, please provide a minimal test or example demonstrating the change.
- Maintain consistent style (follow existing formatting in the repository).

Development workflow
--------------------
1. Create a feature branch.
2. Make changes and run the quick-start example or your unit tests.
3. Open a pull request with a short description and the rationale for the change.

Contact
-------
If you need direct help understanding how the code works, add an issue and include a short reproducible snippet.
